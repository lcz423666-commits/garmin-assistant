```

ICU API ─────┼──► garmin_monitor.py (常驻进程, PID≈3573600)

                → /      → /var/www/garmin-assistant/public/
```

## 七、Web App 功能说明（4 个 Tab）

### Tab 1 · 对话（chat）
- 多轮对话，接入用户当日健康上下文（Garmin + ICU 数据）
- SSE 流式渲染；支持 Markdown 气泡

### Tab 2 · 状态（status）
- 展示当日关键指标：HRV、睡眠评分、静息心率、Body Battery、CTL/ATL/TSB
- 数据来源优先级：ICU 睡眠文件 > Garmin daily JSON
- API: `GET /api/status`

### Tab 3 · 报告（reports）
- 历史报告列表（最近 30 天）
- 点击单日展开全部推送内容（ICU 睡眠分析 + ICU 骑行分析 + Garmin 睡眠晨报 + 运动快报）
- API: `GET /api/reports`、`GET /api/reports/{date}`

### Tab 4 · 路线（route）
- 上传 GPX → 即时解析摘要（里程/爬升/海拔剖面/坡度分布）
- 填写骑行意图（类型/日期/体重）→ LLM 生成路线分析
- 含天气查询（Open-Meteo）
- API: `POST /api/gpx-parse`、`POST /api/gpx-analyze`（SSE）

## 八、ICU 数据流水线

### 8.1 ICU 睡眠分析
```
每轮监控 → ICU Wellness API → select_latest_wellness()
→ extract_sleep_analysis_data() + _enrich_with_garmin()
→ _generate_analysis() [LLM: Kimi-K2.5]
→ _push_message() [PushPlus 微信]
→ _save_state() + 保存 icu_sleep/{date}.json
→ 日志: "ICU 睡眠数据已保存: {path}"
```
**注意**：旧进程（Apr 18 启动）因模块缓存未加载保存逻辑，导致 Apr 21-23 文件缺失，已手动补录。新进程加入 `importlib.reload()` 热更新，后续无需重启即可生效代码变更。

### 8.2 ICU 骑行分析
```
检测新活动 → ICU Activity API + Streams API
→ build_story_points() [拐点地名]
→ fetch_open_meteo_weather() [逐小时气象，使用气温范围区间]
→ build_weather_segments() + summarize_weather_data()
→ _generate_analysis() [LLM]
→ _push_message() + 保存 icu_cycling/{date}_{id}.json
```

## 九、GPX 路线分析流水线

```
POST /api/gpx-parse (UploadFile)
→ _parse_gpx()
    ├── Haversine 距离计算
    ├── 50m 滑动窗口坡度（避免长段平均掩盖陡坡）
    ├── 坡度分布：平坦(≤2%) / 轻松(2-5%) / 中等(5-9%) / 陡坡(>9%)
    └── 海拔剖面下采样（200点）
→ 返回 route JSON（不调 LLM）

POST /api/gpx-analyze (JSON body: route + ride_type + planned_dt + weight_kg)
→ _fetch_weather() [Open-Meteo]
→ LLM prompt（含坡度分布、ride_type 差异化指令）
→ SSE 流式返回 content 事件
```
